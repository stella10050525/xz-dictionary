# XZ Dictionary 图片上传问题说明

## 问题原因

图片上传失败的主要原因是 **localStorage 容量限制**：

### 技术细节
1. **浏览器限制**：大多数浏览器的 localStorage 容量限制为 **5-10MB**
2. **图片编码**：上传的图片会转换为 base64 编码，体积会比原始文件 **增加约 33%**
3. **累积效应**：多张图片累积后很容易超过 localStorage 的容量限制
4. **原代码缺陷**：
   - 单张图片限制为 5MB（太大）
   - 没有捕获 localStorage 写入失败的异常
   - 没有容量检测机制

## 已修复内容

### 1. 降低单张图片大小限制
- **原限制**：5MB
- **新限制**：2MB
- **原因**：base64 编码后约 2.6MB，可存储更多图片

### 2. 增强错误处理
```javascript
try {
    localStorage.setItem('xz_word_images', dataStr);
    // ...
} catch (error) {
    alert('❌ 保存失败！\n\n可能原因：...');
}
```

### 3. 添加容量检测
```javascript
const estimatedSize = new Blob([dataStr]).size;
if (estimatedSize > 4.5 * 1024 * 1024) {
    alert('⚠️ localStorage 容量不足！...');
    return;
}
```

### 4. 详细的错误提示
- 区分文件过大、读取失败、存储失败等不同错误
- 提供具体的解决建议

### 5. 新增存储查看工具
在浏览器控制台输入 `checkStorageUsage()` 可以查看：
- 总存储使用量
- 图片数据占用
- 使用率百分比

## 使用建议

### ✅ 推荐做法
1. **压缩图片**：上传前使用工具压缩（推荐 TinyPNG）
2. **精选图片**：每个单词只能1张图片，选择最能代表该单词的图片
3. **使用链接**：对于网络图片，直接使用图片链接而非上传
4. **定期清理**：删除不常用单词的图片

### ⚠️ 限制说明
- 单张图片：≤ 2MB
- 每个单词：限制 **1张图片**（更换图片会自动替换原有图片）
- 总存储：约 5MB（浏览器限制）

### 🔧 如何清理空间
1. **查看单词图片**：查询单词后点击"清空图片"按钮
2. **清空所有图片**：
   ```javascript
   localStorage.removeItem('xz_word_images');
   ```
3. **查看存储使用**：
   ```javascript
   checkStorageUsage();
   ```

## 错误提示说明

### "大小超过2MB"
- **原因**：单张图片文件超过 2MB
- **解决**：压缩图片后重新上传

### "localStorage 容量不足"
- **原因**：总存储空间接近 5MB 限制
- **解决**：删除一些旧图片或清空图片数据

### "localStorage 容量已满"
- **原因**：写入时超过浏览器限制
- **解决**：
  1. 清空部分图片数据
  2. 使用图片链接代替上传
  3. 退出浏览器隐私模式

## 查看控制台日志

按 `F12` 打开开发者工具，在 Console 标签可以看到：
- 具体的错误信息
- localStorage 写入失败的原因
- 存储使用情况（运行 `checkStorageUsage()`）

## 替代方案

如果频繁遇到存储限制，可以考虑：

### 方案 1：使用外部图床
上传图片到图床（如 Imgur、阿里云 OSS），然后使用图片链接

### 方案 2：使用 IndexedDB
未来版本可以考虑升级到 IndexedDB，容量限制更大（通常 50MB+）

### 方案 3：后端存储
部署后端服务，将图片存储在服务器上
